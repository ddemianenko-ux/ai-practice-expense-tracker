# Рефлексія та критичний аналіз (Reflection)

## Що спрацювало
* **Швидкий старт:** ChatGPT ідеально підійшов для створення boilerplate-коду (функції роботи з JSON). Це зекономило багато часу на рутинних операціях.
* **Автодоповнення команд:** GitHub Copilot чудово впорався з генерацією об'єктів `argparse.ArgumentParser`. Він швидко зрозумів контекст з назв функцій.

## Аналіз згенерованого коду та правки
1. **Збереження даних (ChatGPT):**
   * *Запропоновано:* Функція збереження `json.dump(data, f)`.
   * *Змінено:* Я додав параметри `indent=4` та `ensure_ascii=False`.
   * *Чому:* Щоб JSON файл був читабельним для людини і правильно відображав кирилицю (українську мову) у описах витрат.
2. **Аргументи CLI (GitHub Copilot):**
   * *Запропоновано:* Скорочені аргументи `-c`, `-a`.
   * *Змінено:* Я залишив лише явні параметри `--category`, `--amount`, `--desc`.
   * *Чому:* Для кращої читабельності та уникнення плутанини під час тестування інструменту. 

## Загальний висновок
Використання АІ не замінює розуміння коду, але ефективно знімає "страх чистого аркуша" та пришвидшує написання синтаксичних конструкцій. Важливо завжди робити code review згенерованих фрагментів.
